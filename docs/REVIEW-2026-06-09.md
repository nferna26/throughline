## P1 — must fix before launch

### 1. "Drag in DRM-free EPUBs" is not implemented
There is no drag-and-drop import anywhere: no drop handler in the frontend, no Tauri
drag-drop event listener, no `fileAssociations` in `src-tauri/tauri.conf.json`. The only
import paths are the file-picker dialog (`src/App.tsx:104`) and Discover downloads. A
reader who drags an EPUB onto the window gets silence — the drop is swallowed.
This is a public marketing claim (CLAUDE.md:105 "drag in DRM-free EPUBs") **and the first
link of the golden loop** (CLAUDE.md:82 "Discover or drag in a book").
**Smallest fix:** listen for the webview drag-drop event (`getCurrentWebview()
.onDragDropEvent`) in `App.tsx`, filter to `.txt`/`.epub`, and funnel paths into the
existing `cmd_import_book` → same `ImportOutcome` routing as the picker.

### 2. AI-history retention and "Forget now" break permanently after the first aged cloud call
`ai_request_usage.request_id` has a foreign key to `ai_requests(id)` with no cascade
(`src-tauri/src/migrations.rs:297`), and every open sets `PRAGMA foreign_keys = ON`
(`src-tauri/src/db.rs:25`). Every cloud Ask writes a usage child row
(`src-tauri/src/commands/ai.rs:531-534`). The retention sweep deletes parents only
(`src-tauri/src/ai_retention.rs:34-39`). Once any unsaved cloud ask ages past the window
(default 90 days), the DELETE hits the FK and **the whole statement aborts — nothing is
ever swept again**, and the reader-facing "Forget now" button (Settings → Request history)
returns `FOREIGN KEY constraint failed`. Reproduced against the exact production schema
in SQLite: one usage-bearing row blocks deletion of *all* eligible rows.
The suite can't see it: the sweep test hand-rolls a schema with no usage table and no FK
pragma (`ai_retention.rs:47-56`).
**Smallest fix:** in `sweep()`, delete matching `ai_request_usage` rows first (same
predicate via subquery), and rewrite the test to use `migrations::apply_pending` with
`foreign_keys = ON`.

### 3. Deep Study briefing cache violates the counsel-reviewed copyright posture
CLAUDE.md §3 (binding, 2026-06-08): briefings "stay … **non-persistent unless saved**."
The briefing — AI output derived from section text — is cached in `localStorage` with no
TTL and survives restarts without any save action (`src/sectionBriefing.ts:62-86`, written
at `src/components/SectionBriefingCard.tsx:151-153`). Root cause: AGENTS.md's older AI
contract explicitly *required* a cache ("must be cached, dismissable, regenerable"), and
CLAUDE.md's newer counsel language supersedes it — the two binding-claiming docs now
disagree.
**Smallest fix:** make the cache session-scoped (module-level in-memory map) — that still
satisfies AGENTS.md's anti-respam intent within a sitting while restoring
"non-persistent unless saved" — and reconcile the AGENTS.md AI-contract paragraph with
CLAUDE.md (see P2-13).

### 4. The first screen overpromises: "Your books and notes never leave this Mac"
Welcome card, `src/screens/Today.tsx:108` (pinned by `e2e/walkthrough.spec.ts:26`). Since
the AI pivot, both shipped configurations send the reader's **selected passage** (and, in
Deep Study, the section) to a cloud provider once the reader opts in. The absolute claim
on the very first screen contradicts the app's own consent sheet ("Your selected passage
goes to {host}") and the Settings trust card, which are carefully honest. A launch-day
reviewer will lead with this.
**Smallest fix:** reword to match the trust card's precision — e.g. "Your books stay on
this Mac. If you ask the tutor, only the passage you select is sent — never the book." —
and update the e2e assertion in the same commit.

## P2 — a reader would feel it

### 5. Pausing a plan doesn't stop the Today pace clock
`cmd_pause_plan` sets `lifecycle='paused'` but never touches `status`
(`src-tauri/src/commands/plans.rs:139-149`), while pace/forecast gating keys on `status`
(`src-tauri/src/plan.rs:257`). `fetch_plan_for_book` falls back to the paused plan when no
active one exists (`src-tauri/src/commands/db_helpers.rs:204-208`), so a reader who pauses
their only plan watches "Behind · N days" **grow during the pause**, recovery panel
included — the exact shame the pause feature promises to prevent. (The TS type already
anticipates a `"paused"` status: `src/types.ts:33`.)
**Smallest fix:** have `cmd_pause_plan` also set `status='paused'` (resume restores
`'active'`), and treat any non-active lifecycle as not-pacing in `plan::compute`.

### 6. A book whose last plan is let go becomes permanently unreachable
"Let go" is offered on archived/paused plans (`src/components/PlansView.tsx:206`). Archive
the only plan, then let it go: `cmd_today` returns `None` (book exists, no plan —
`src-tauri/src/commands/books.rs:180-182`), and the app renders the first-run welcome card
with **no BookSwitcher, no Notes tab, no path to the book** (`src/App.tsx:259-261`).
Re-importing the same file dedups onto the plan-less book and lands back on the welcome
card. The library and notes are intact on disk but unreachable without DB surgery.
**Smallest fix:** in `cmd_today`, when an active book exists with no live plan, return a
card in a "no plan" state (Today already has plan-empty copy in PlansView) — or have the
welcome branch check `cmd_list_books` and offer "Pick up where you left off" →
`cmd_start_new_plan`.

### 7. Failure copy assumes LM Studio — the paid reader's first error is a dead end
`humanizeError` maps any connection-ish failure to "Can't reach the local model server.
Is LM Studio (or Ollama) running on this Mac?" regardless of provider
(`src/components/MarginTutorCard.tsx:125-134`, duplicated at
`src/components/SectionBriefingCard.tsx:57-65`). A company-mode reader offline gets
LM Studio jargon — precisely the AI-cold-start-cliff CLAUDE.md names. The same function
says "Check Settings → AI" — **no such section exists** (it's "Assistance",
`src/screens/Settings.tsx:210`).
**Smallest fix:** thread the known `provider` into `humanizeError` ("Can't reach
Throughline AI — check your connection and try again."), fix "Settings → Assistance", and
de-duplicate the helper.

### 8. Company gauge shows "Almost out" when the proxy is merely unreachable
`CompanyPanel.fuel()` returns `{label: "Almost out", level: "empty"}` for `credits ==
null` **or any non-"active" status** — including `"unknown"`, which is what a network
blip or proxy 5xx produces (`src/components/CompanyPanel.tsx:7-13`;
`cmd_company_credits` ignores HTTP status, `src-tauri/src/commands/ai.rs:857-864`). A paid
reader with momentary Wi-Fi trouble is told their allowance is nearly spent — a false
alarm with purchase implications. (`TutorFuel` handles the same case correctly by hiding,
`src/components/TutorFuel.tsx:36`.)
**Smallest fix:** distinguish `status === "active"` (real gauge) from unknown/unreachable
("Can't check right now — the tutor still works") in `fuel()`.

### 9. Two recovery options are placebo buttons
"Add 10 min for the next N sessions" and "Catch up on the weekend" only set a transient
message — no state changes, nothing persists, nothing is remembered tomorrow
(`src/screens/Today.tsx:338-347`). The buttons read as commitments ("Plan: add 10 min…")
and silently evaporate on reload. ExtendFinish is the only option that does anything.
**Smallest fix:** either make them real (bump `reading_rhythm_minutes` for N sessions; a
stored marker the recap acknowledges) or reword them as advice rather than actions ("Try
adding 10 minutes today") with no fake "Plan:" confirmation.

### 10. Paid-but-unactivated readers have no activation door at the moment of intent
If a buyer never clicks the activation deep link, their first lens click opens
`AiSetupSheet` offering only Paste-key / LM Studio / Copy-prompt
(`src/components/AiSetupSheet.tsx:261-280`). The "Already bought? Paste your activation
code" entry exists only in Settings → CompanyPanel (`src/components/CompanyPanel.tsx:114`).
The zero-setup promise depends on a deep link that can be missed; the recovery is a
Settings detour the sheet was designed to eliminate.
**Smallest fix:** add a quiet "Already bought Throughline AI? Enter your code" row to the
sheet's `not_connected` state, reusing `cmd_activate_company`.

### 11. Activation feedback is nearly invisible — at the $20 moment
The deep-link handler swallows failures entirely (a bad/expired token does nothing
visible) and on success only refreshes Today with no confirmation (`src/App.tsx:79-92`).
The buyer's highest-anxiety moment ("did my purchase take?") has no acknowledgment unless
they happen to open Settings → Assistance → Throughline AI.
**Smallest fix:** on `tl-activate` success show a one-line confirmation ("Throughline AI
is active — ask the tutor anything"); on failure surface the existing "invalid, expired,
or already used" message.

### 12. `rm -rf` printed in reader-facing Settings
"Rollback: `rm -rf ~/Library/Application Support/Throughline`" renders as a hint under
Storage (`src/screens/Settings.tsx:205`). The experience bar bans plumbing words anywhere
a reader can see; a copy-pasteable destructive shell command is the worst case of it (and
README already documents the same rollback for developers, where it belongs).
**Smallest fix:** delete the hint (or replace with a "Reveal in Finder" button).

### 13. AGENTS.md still binds agents to the pre-pivot AI contract
AGENTS.md calls itself "the binding contract" and still says remote endpoints are refused
by default, the briefing is "local-only", and AI "never calls a remote endpoint by
default" (AGENTS.md "AI contract"). CLAUDE.md (newer, and the stated winner) ships cloud
tutoring as a feature. Future agent sessions reading AGENTS.md (the codex/* workflow does)
will enforce dead rules or, worse, treat live code as violations.
**Smallest fix:** rewrite AGENTS.md's AI-contract and posture sections to defer to
CLAUDE.md, including the briefing-persistence language (see P1-3).

### 14. Codex token refresh can strand both the app and the reader's CLI
Refresh rotates the `refresh_token`, then the write-back to `~/.codex/auth.json` is
discarded on failure: `let _ = write_codex_auth(&auth);`
(`src-tauri/src/ai_providers.rs:721-730`). If that write fails (permissions, disk), the
old refresh token is already consumed — the app *and* the user's separate Codex CLI are
left with revoked credentials and no signal why.
**Smallest fix:** propagate the write error into the call's error path ("Signed out —
sign in again") instead of silently continuing, and only rewrite the CLI file when the
creds came from it (already the case) — but loudly.

### 15. Raw provider bytes can reach the reader's error card
A malformed SSE line surfaces as `bad SSE chunk: parsing SSE chunk: {payload}` — the raw
payload, which mid-stream contains tutor output quoting the reader's selection
(`src-tauri/src/ai_client.rs:219-220`, `:366-377`); `humanizeError` passes unknown
messages through. Verified this never reaches `app.log` (the log status is a constant —
`commands/ai.rs:541-549`) and the frontend has zero `console.*` calls, so it is a
reader-facing-jargon problem, not a log leak — but it is still book-derived text inside an
error message.
**Smallest fix:** truncate the error to a generic "The answer stream was interrupted —
try again" at the `parse_sse_data_line` call site and keep the payload out of the message.

## P3 — quality, copy, and dead weight

16. **UTC day boundaries.** All plan/streak day math uses UTC (`plan.rs:215` via
    `Utc::now().naive_utc().date()`, streaks via `DATE('now')`,
    `commands/books.rs:613`). For a US evening reader, "today" flips at 4–8 pm local:
    finish tonight's section at 9 pm ET and Today immediately shows tomorrow's. Fix: use
    local dates at the day-boundary call sites.
17. **Migration transactionality is documented but absent.** `apply_pending`'s doc
    promises "up + record … inside one transaction" (`migrations.rs:16-17`); the code has
    no transaction (`:98-114`). Safe today only because every body is idempotent. Wrap
    each migration+record in a transaction (or fix the comment).
18. **Export-path banned-roots check is exact-match only.** `"/etc/"` (trailing slash),
    `"/etc/sub"`, `"/Users"` all pass (`settings.rs:221-228`). Self-foot-gun only;
    normalize + `starts_with` on the banned list.
19. **Book path/title in stderr.** `cmd_import_book` eprintln!s the full source path and
    title (`commands/books.rs:25`, `:65-70`); GUI-app stderr lands in the macOS unified
    log (sysdiagnose-collectable). Titles are content-adjacent metadata. Drop path/title
    from stderr (ids are enough); generally, migrate stray `eprintln!` to `tracing`.
20. **Company "Test connection" doesn't test.** Static success string
    (`ai_providers.rs:1077-1081`) even with the relay down. Probe `/v1/credits` instead.
21. **`~/GBrain/Reading/*` is created on every launch**, before any export and even when a
    custom export root is configured (`paths.rs:73-81` called from `db.rs:13`) — a mystery
    folder appears in `$HOME` on first run. Export already creates dirs on demand
    (`export.rs:34-39`); drop the launch-time creation.
22. **Unbounded `app.log`** (`rolling::never`, `log.rs:43`). Use daily rolling or a size
    cap.
23. **Stale `ai_local_only`.** The DTO field still claims "HARD privacy invariant"
    (`settings.rs:137-138`) but nothing consults it at call time (provider is
    authoritative; Local is hardwired loopback at `commands/ai.rs:405-407` and
    `settings_cmds.rs:141-148`). No UI renders it. Either delete the field+key or fix the
    comments.
24. **Plan-completion UPDATE hits soft-deleted/archived plans**
    (`commands/sessions.rs:188-193` has no lifecycle/deleted filter).
25. **`cmd_update_note` COALESCE can never clear a field** (short_quote/anchored_text;
    `commands/notes.rs:62-68`). Matters only if the UI ever offers removal.
26. **Token/endpoint/locator jargon in reader-visible copy.** "price per million tokens"
    (`Settings.tsx:305`), "Estimated from token usage" (`AiUsageCard.tsx:43`),
    "OpenAI-compatible local endpoint" (`Settings.tsx:263`), raw `char:48211` locators in
    the New-note modal (`TextReader.tsx:1085`) and the tutor quote chip
    (`MarginTutorCard.tsx:463`). The BYO surfaces can justifiably name API keys; tokens
    and raw locators have plain-language alternatives ("per million words of input/output",
    chapter + "% in").
27. **Company readers see the BYO spend card.** `AiUsageCard` renders unconditionally
    (`Settings.tsx:344`): company calls appear as `company: $0.00 · N` chips (company has
    no catalogued price), plus a monthly-cap input that doesn't apply to them. Hide or
    reframe when the provider is company ("included in your purchase").
28. **Cloud-consent copy says "under your API key" for every provider**
    (`MarginTutorCard.tsx:686`) — wrong for Codex (ChatGPT login) and company (no key).
29. **Anthropic-arm SSE parser silently ignores malformed `data:` lines**
    (`ai_providers.rs:389-391`) — a corrupted stream truncates the answer and still ends
    "done"; the OpenAI path hard-errors. Surface it.
30. **Circuit breaker asymmetry.** Only `run_chat_call` (Local + OpenAI) checks the
    breaker (`ai_client.rs:277-280`); Anthropic/Codex/Company never fail fast, and the one
    global breaker is shared across providers (LM Studio failures briefly block OpenAI).
31. **`.txt` import has no size cap** (`import.rs:632` reads the whole file; EPUB
    extraction is capped at 500 MB, and even there a single huge member is read into
    memory before the accumulated check, `import_epub.rs:265-296`). Self-DoS only.
32. **Silent failure paths in modals.** NotePanel's save shows nothing on error
    (`TextReader.tsx:1052-1068`); App's import alert falls back to `JSON.stringify(e)`
    (`App.tsx:121`). Route both through `errorMessage()`.
33. **`AiSetupSheet.checkLocal` persists the default base URL as a side effect of opening
    the LM Studio panel** (`AiSetupSheet.tsx:186`), clobbering a custom local port.
34. **Dead weight.** `src/screens/AiPanel.tsx` (232 lines, Shot-4 era) and
    `src/screens/Onboarding.tsx` (33 lines) are imported nowhere;
    `plan::pace_state`/`expected_completed` are test-only (`plan.rs:99-133`); legacy
    `rg.*` localStorage keys and the `.rgtmp` temp suffix (`ai_providers.rs:605`) predate
    the rename. Delete the screens; keep or wire the rest deliberately.
35. **`COMMAND_API_VERSION` history is incomplete** — constant is 5, the mandated history
    documents only →2 and →3 (`lib.rs:66-72`).
36. **Plan purge leaves exported mirrors.** The 30-day "let go" sweep hard-deletes notes
    rows but not their Markdown files (`plans.rs:211-231`) — defensible
    (Markdown-as-canonical) and the toast says "notes kept for 30 days", but the exported
    files outliving the in-app copies deserves a deliberate yes.

## Pass 2 — the stranger's walk (what holds up)

The cold-open path passes the two-minute test on code inspection: welcome card (one
primary action) → Discover shelves paint instantly from the bundled seed → "Get" →
"Saved to your library" → Book Setup Sheet with an honest feasibility estimate and a
"Decide later" exit → Today with "Plan ready. You are not behind." → reader → select →
note saved → Markdown exported. Empty states teach the next step everywhere I traced
(welcome, notebook, margin, Discover no-matches, plans-resting). Errors are
overwhelmingly in the app's voice with a next step; the exceptions are P2-7 and P3-32.
Every modal uses a real focus trap (`useDialog`), Escape works everywhere it should, axe
runs in CI with a serious/critical floor.

The tutor experience is the strongest part of the app: consent at the moment of intent
with provider-truthful copy, the passage quoted above every answer (the "quotes your
selection" promise, `MarginTutorCard.tsx:457-464`), brief→deeper tiering, the dignified
copy-prompt fallback with scaffolding stripped (tested), the three-door cap screen with
the free path first, and an audit trail (Settings → Request history) that shows
"Local" vs "Sent → host" per request — the posture made visible.

## What I tried to break and couldn't (verified-strong)

- **Loopback enforcement**: `validate_base_url`/`is_loopback_host` reject public DNS,
  RFC-1918, 0.0.0.0, the metadata IP; Local is hardwired `local_only=true` at both the
  command gate and the dispatch arm; the only user-settable URL slot is loopback-validated
  at save; cloud endpoints are code constants; `tauri-plugin-http`/`shell` are banned by a
  test; CSP `connect-src 'self' ipc:` means the webview can't reach the network at all.
- **Secrets**: Keychain-only via `keyring` with an in-memory session cache; only
  `key_present` booleans cross IPC; never logged; presence flags avoid launch prompts.
- **Prompt hardening**: static fence + zero-width marker neutralization + safety preamble
  on every mode; preview==sent pinned byte-for-byte; selection capped at 2k chars (6k for
  briefings) — the whole book cannot be sent; reader-facing fallback prompt carries no
  scaffolding. All tested.
- **Relay client**: same Anthropic body to `/v1/tutor`, Bearer license, model locked to
  Sonnet server-side of the client (`settings.rs:330` ignores any stored model), 402 is a
  typed cap signal, dollars never reach company-mode refusals, the support mailto is a
  compile-time constant tested content-free.
- **Imports**: SHA-256 dedup *before* any copy; sources chmod 444; DRM markers refuse
  (never parse around); no zip-member path extraction (no zip-slip surface); Gutendex
  downloads are https-only against a dot-boundary host allowlist with redirect
  re-validation and a streamed 64 MB cap.
- **Data layer**: WAL + `synchronous=NORMAL` (ADR'd and pinned), idempotent append-only
  migrations tested against legacy DBs, corrupt-DB recovery preserves the old file,
  path-traversal guard on `book_id`, atomic exports with fsync + tmp cleanup, and the
  bin-guardrail (runtime panic + source-scan test) that keeps diagnostics off the real DB.

## Verdict: **don't ship yet** — but this is days of work, not weeks.

The architecture is genuinely trustworthy: the privacy machinery (loopback gate, Keychain,
fences, audit trail, export hygiene) is real, tested, and consistent, and the experience
writing is among the best I've reviewed. What blocks launch is narrower: one advertised
capability doesn't exist, one privacy mechanism breaks permanently on a timer, one binding
legal posture is violated by a cache, the first screen says something the product no
longer means, and the paid build's failure path talks to the wrong customer.

**The five fixes that must happen first:**

1. **Implement drag-and-drop import** (P1-1) — it is a public claim and the golden loop's
   first link; a webview drag-drop listener into `cmd_import_book` closes it.
2. **Fix the retention sweep FK** (P1-2) — delete usage children in the sweep and add a
   regression test on the real migrated schema; this also un-breaks the reader-facing
   "Forget now" button.
3. **Make the briefing cache session-only** (P1-3) and reconcile AGENTS.md to CLAUDE.md
   (P2-13) so the counsel-reviewed "non-persistent unless saved" holds and future agent
   sessions stop reading a dead contract.
4. **Tell the truth at first contact** (P1-4 + P2-7): reword the welcome promise (and its
   e2e pin), make error copy provider-aware, and fix "Settings → AI" → "Assistance" — the
   paid reader's first failure must not be an LM Studio dead end.
5. **Close the plan-state traps** (P2-5 + P2-6): pause must stop the Today clock, and a
   book with no live plan must keep a visible way back.

Everything else on this list can follow launch without breaking trust — these five can't.
